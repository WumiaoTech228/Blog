# 这个博客是怎么搭起来的

这是这个博客的第一篇正式记录。与其写一篇“欢迎来到我的博客”，不如直接把它当成一次建站复盘：这个站为什么这么搭、现在解决了什么问题、后面还准备补什么。

## 为什么选 Astro

我想要的是一个长期写作的地方，而不是一个需要经常维护服务器的系统。Astro 比较适合这个目标：

- 页面可以静态生成，访问速度快。
- Markdown 内容天然适合写技术笔记。
- 想加组件时也能接 React，但普通页面不需要把整套前端运行时塞给用户。

对个人博客来说，这个取舍很实用。大部分页面只是阅读内容，不需要复杂的客户端状态。

## 为什么用 Keystatic

直接写 Markdown 没问题，但有时我也希望能用一个可视化后台快速补文章、改摘要、上传封面。Keystatic 的好处是它没有把内容锁进数据库，而是继续把文章存在 Git 仓库里。

这样一来，内容还是文件，版本历史也还在 Git 里。后面如果要迁移、重构或者换 CMS，成本不会太高。

## 为什么部署到 Cloudflare Pages

这个站目前没有动态后端，部署到 Cloudflare Pages 基本够用：

- 静态资源走边缘网络。
- 不需要单独买服务器。
- 和 GitHub 仓库联动后，推送代码就能触发部署。

对我这种主要写笔记、记录项目进展的站点来说，稳定和少维护比功能复杂更重要。

## 接下来我准备写些什么？

短期会先补这些内容：

1. 这个博客从脚手架到上线的完整过程。
2. LepoProxy 的设计思路和开发日志。
3. AI 编程工具的实际体验，包括哪里省时间、哪里容易出错。
4. 一些日常开发里踩过的坑。

这个站点本身也会继续迭代。第一版先把写作、归档、标签、RSS 和基础 SEO 做稳，后面再考虑搜索、系列文章和更细的项目页面。
